# Контракт: палитра бренда SUMMY

Цвета читают **все продукты** (сайт, кабинет, adminapp, будущий clientapp),
поэтому канон здесь. В продуктах — копии этих значений с тестом-пином,
который не даёт цвету уехать незаметно (менять можно, но осознанно и через
ревью + правку этого файла).

## Токены (10 цветов)

| Токен | HEX | Роль |
|---|---|---|
| `summy-green` | `#8c8253` | олива — главный акцент |
| `summy-green-dark` | `#6e6741` | олива тёмная |
| `summy-beige` | `#f2e9da` | беж — фон секций (витрины) |
| `summy-beige-light` | `#faf6ee` | беж светлый |
| `summy-sand` | `#d3b89d` | песок |
| `summy-graphite` | `#3c3c3b` | графит — текст |
| `summy-grey` | `#cfc9be` | серый тёплый |
| `summy-sky` | `#6995b5` | небо — информация |
| `summy-sun` | `#f5c74d` | солнце — внимание/предупреждение |
| `summy-white` | `#ffffff` | белый |

Кабинет мастера исторически живёт на своём оттенке акцента
(`--plum: #50603a`, болотный) — легаси до общего редизайна, новое на него
не равнять.

## Правило «мебель нейтральная» (рабочие интерфейсы)

В **рабочих** интерфейсах (кабинет, adminapp, clientapp — на них смотрят весь
день): беж и песок **сплошной заливкой не идут** — только акценты и
разделители; фон нейтральный; олива — единственный акцент. Проверено на живом
экране adminapp (ADR-010): бежевая заливка «топит» рабочий экран.

На **витринах** (сайт) беж как фон секции — норма, брендбук §2.7 писан для них.

## Сигнальный цвет

Красного в палитре нет; «плохо/просрочено» сейчас графитовое. Открытый вопрос
владельцу (бренд-решение): завести один сигнальный (предложение — приглушённый
кирпичный, дружит с оливой). Правится одним токеном `--destructive`.
